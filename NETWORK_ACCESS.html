üè¢ Same Network (LAN) - Detailed Guide

Step 1: Start the Server

Run production build
npm run prod

The server will start on port 5000 and listen on all network interfaces (0.0.0.0)

Step 2: Find Your Server's IP Address

Windows:

Open Command Prompt and run:
ipconfig

Look for "IPv4 Address" under your active network adapter (e.g., 192.168.1.100)

Mac/Linux:

Run:
hostname -I

Step 3: Access from Another Machine

On any device connected to the same network, open a browser and visit:

Replace YOUR_IP with your server IP
http://YOUR_IP:5000
Example: If your server IP is 192.168.1.100, visit
http://192.168.1.100:5000

üåç Internet Access with ngrok

For accessing the app from different networks or the internet, use ngrok (free):

Setup ngrok:

Download ngrok
Go to: ngrok.com/download
Create free account
Signup at: dashboard.ngrok.com
Copy your auth token
From dashboard: Settings ‚Üí Auth Token
Configure ngrok
ngrok config add-authtoken YOUR_AUTH_TOKEN
Start server & ngrok
Terminal 1: npm run prod
Terminal 2: ngrok http 5000
Share the link
ngrok shows a URL like: https://abc123.ngrok.io
Anyone can access your app with this link!
‚úÖ ngrok creates a free public tunnel to your local server

‚öôÔ∏è Environment Configuration

Create a .env file to configure your server for different networks:

.env file example
NODE_ENV=production BACKEND_PORT=5000 VITE_API_URL=http://192.168.1.100:5000

Configure for different scenarios:

Local: VITE_API_URL=http://localhost:5000
LAN (192.168.1.100): VITE_API_URL=http://192.168.1.100:5000
ngrok tunnel: VITE_API_URL=https://abc123.ngrok.io
Cloud (example.com): VITE_API_URL=https://example.com
‚ÑπÔ∏è After changing .env, rebuild:

üî• Firewall Configuration

Ensure port 5000 is allowed through your firewall:

Windows:

PowerShell (Admin)
netsh advfirewall firewall add rule name="Quiz App" dir=in action=allow protocol=tcp localport=5000

Linux (UFW):

Terminal
sudo ufw allow 5000/tcp

macOS:

System Preferences ‚Üí Security & Privacy ‚Üí Firewall Options ‚Üí Add port 5000

üêõ Troubleshooting

‚ùå Can't connect from another machine

Check:

Server is running: npm run prod
Both machines on same network (check WiFi)
Correct IP address (run ipconfig)
Firewall allows port 5000
Try ping: ping SERVER_IP

‚ùå Port 5000 already in use

Windows - Kill process
netstat -ano | findstr :5000 taskkill /PID /F

‚ùå Frontend not loading data

Check .env - VITE_API_URL must match your server's actual IP/URL
